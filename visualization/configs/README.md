# Visualization Configuration Files

This directory contains YAML configuration files for the visualization pipeline.

## Available Configurations

### Source 1 Experiments

**`source_1_original.yaml`**
- ResNet50 experiments on source_1 dataset
- **Without data augmentation**
- 4 split strategies: batch, plate, well, random (3 runs each)
- Usage: `python -m masterthesis.visualization --config visualization/configs/source_1_original.yaml`

**`source_1_w_augmentation.yaml`**
- ResNet50 experiments on source_1 dataset  
- **With data augmentation**
- 4 split strategies: batch, plate, well, random (3 runs each)
- Usage: `python -m masterthesis.visualization --config visualization/configs/source_1_w_augmentation.yaml`

## Configuration Structure

Each YAML file contains:

```yaml
source: source_1                    # Dataset identifier
architecture: ResNet50_Modified     # Model architecture
augmentation: true/false            # Data augmentation flag

mappings:                           # JSON encoding files
  labels: source_1_encoding_pert_iname.json
  plate_ids: source_1_encoding_Metadata_Plate.json
  well_ids: source_1_encoding_Metadata_Well.json
  batch_ids: source_1_encoding_Metadata_Batch.json

experiments:                        # Checkpoint paths by split strategy
  - split_name: batch
    npy_paths: [...]
  - split_name: plate  
    npy_paths: [...]
  - split_name: well
    npy_paths: [...]
  - split_name: random
    npy_paths: [...]

plot_settings:                      # Visualization settings
  figsize: [12, 12]
  dpi: 150
  show_replicate_points: true
  show_std: true
```

## Split Strategies

- **batch**: Data split by experimental batch
- **plate**: Data split by plate identifiers
- **well**: Data split by well positions  
- **random**: Random data splitting

Each strategy contains 3 independent runs for statistical analysis.

## Path Format

All checkpoint paths are relative to the `checkpoints/` directory:
```
checkpoints/ResNet_source_1_original/{run_name}/Test_evaluation_results.npy
checkpoints/ResNet_source_1_w_augmentation/{run_name}/Test_evaluation_results.npy
```

## Notes

- Configuration files were auto-generated by scanning checkpoint directories and reading `split_keys.json` files
- All paths have been validated to ensure files exist
- JSON mapping files are located in the project root directory
- Each `.npy` file contains test evaluation results for visualization analysis
